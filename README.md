# Share Basket Server

買い物リストを共有できるアプリケーションのバックエンドサーバー

## アーキテクチャ

### レイヤー全体像

```
.
├── domain/          # ドメインレイヤー
│   ├── account.go
│   ├── account_id.go
│   ├── personal_shopping_item.go
│   ├── transaction.go
│   └── user.go
│
├── usecase/         # ユースケースレイヤー
│   ├── login.go
│   ├── signup.go
│   └── ...
│
├── presentation/    # プレゼンテーションレイヤー
│   ├── handler/     # HTTPハンドラー
│   ├── response/    # レスポンス生成
│   ├── validator/   # リクエストバリデーション
│   └── server/      # HTTPサーバー設定
│
└── infra/          # インフラストラクチャレイヤー
    ├── aws/        # AWS関連の実装
    └── rdb/        # リレーショナルデータベース関連の実装
```

### レイヤーの説明

#### ドメインレイヤー (`domain/`)
- ビジネスロジックの中核
- エンティティとその振る舞いを定義
- 外部依存を持たない純粋なビジネスロジック
- リポジトリインターフェースの定義

#### ユースケースレイヤー (`usecase/`)
- アプリケーションのユースケースを実装
- ドメインレイヤーを使用してビジネスロジックを実行
- 入力ポートと出力ポートを定義
- トランザクション管理

#### プレゼンテーションレイヤー (`presentation/`)
- HTTPリクエスト/レスポンスの処理
- リクエストのバリデーション
- ユースケースの呼び出し
- レスポンスの生成

#### インフラストラクチャレイヤー (`infra/`)
- 外部サービスとの連携（AWS Cognito等）
- データベースアクセス（RDB）
- 外部APIとの通信
